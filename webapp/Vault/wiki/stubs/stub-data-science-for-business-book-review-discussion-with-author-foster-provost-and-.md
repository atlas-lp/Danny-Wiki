## Abstract

This is a recorded discussion from the Product Book Club (2022-07-25) featuring Foster Provost, co-author of "Data Science for Business," in conversation with product managers. The discussion focuses on evaluating data science projects, calculating expected ROI for data-science-heavy initiatives, and collaborating effectively with data scientists. Provost introduces multiple ways to use the expected value framework for decision-making in predictive analytics and hints at a fourth approach planned for a second edition. The conversation bridges the gap between data science and product management, framing the book's concepts through a product lens.

## Key Claims

- The expected value framework is a central tool for evaluating data science initiatives and calculating expected ROI for DS-heavy projects
- There are at least three ways (presented in the book) to use the expected value framework, with a fourth approach in development for a potential second edition
- Expected value helps determine which machine learning algorithm is better for a given problem and whether machine learning is even needed
- Probabilistic nature of predictive analytics means you cannot simply assign a fixed value to outcomes; the expected value framework addresses this uncertainty
- The expected value framework supports "analytic engineering" by decomposing problems into values and probabilities, enabling structured application of machine learning
